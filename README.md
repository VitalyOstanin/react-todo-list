## Описание

Учебный pet project Todo List - современное SPA приложение для управления задачами, построенное на React с TypeScript. Приложение поддерживает локализацию на русском и английском языках, имеет мок API с возможностью перехода на реальный бэкенд.

## Технологический стек

- **React 19** - библиотека для создания пользовательских интерфейсов
- **TypeScript** - типизированный JavaScript
- **Vite** - быстрый сборщик и dev-сервер
- **React Router** - маршрутизация для SPA
- **i18next** - интернационализация
- **Axios** - HTTP клиент
- **Jest** - фреймворк для тестирования
- **Testing Library** - утилиты для тестирования React компонентов

## API

Приложение использует мок API, который можно легко заменить на реальный бэкенд. Для переключения измените константу `USE_MOCK_API` в файле `src/services/todoService.ts`.

### Эндпоинты

- `GET /api/todos` - получение всех задач
- `POST /api/todos` - создание новой задачи
- `PUT /api/todos/:id` - обновление задачи
- `DELETE /api/todos/:id` - удаление задачи

### Типы данных

```typescript
interface Todo {
  id: string;
  title: string;
  completed: boolean;
  createdAt: Date;
  updatedAt: Date;
}
```

## Локализация

Приложение поддерживает русский и английский языки. Русский язык установлен по умолчанию. Переключение языка доступно через кнопку в интерфейсе.

Файлы локализации находятся в папке `src/locales/`.

## Разработка

### Добавление новых переводов

1. Добавьте ключи в файлы `src/locales/ru/translation.json` и `src/locales/en/translation.json`
2. Используйте хук `useTranslation` в компонентах

### Переход на реальный API

1. Измените `USE_MOCK_API` на `false` в `src/services/todoService.ts`
2. Настройте `API_BASE_URL` для вашего бэкенда
3. При необходимости адаптируйте типы данных в `src/types/todo.ts`

### Добавление новых компонентов

1. Создайте компонент в папке `src/components/`
2. Добавьте соответствующие тесты в `src/components/__tests__/`
3. Экспортируйте компонент для использования в других частях приложения
